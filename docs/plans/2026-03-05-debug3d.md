# Debug 3D — Consciência Espacial para o Claude

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Criar um sistema de 3 peças que dá ao Claude visibilidade real da cena 3D para eliminar bugs de posicionamento (objetos dentro de outros, coordenadas em world-space vs local-space, etc).

**Architecture:**
- **Peça 1:** `Debug3D.tsx` — componente runtime ativado via `?debug=1` que mostra eixos, bounding boxes e expõe `window.__cena3d` com JSON completo da hierarquia
- **Peça 2:** Skill local `3d-posicoes` — muda o comportamento do Claude para sempre pedir o JSON antes de mexer em posições
- **Peça 3:** Comentários estruturados de hierarquia nos `.tsx` existentes — contexto estático legível

**Tech Stack:** React 19, @react-three/fiber, @react-three/drei (Helper, Grid já instalados), Three.js 0.183, TypeScript 5.9

## Contexto da cena

O quarto tem dois Canvas independentes (`tela-quarto` e `tela-interacao`).

Hierarquia de grupos relevante no Canvas do quarto:
```
<Canvas>                                    ← world root
  <Quarto position={[0, -2, 0]}>           ← OFFSET GLOBAL: Y=-2
    <Escrivaninha position={[-2.5,2,-3]}>  ← local; world = [-2.5, 0, -3]
      <Banquinho position={[0,-0.35,2.4]}> ← local; world = [-2.5, -2.35, -0.6]
    <Cama position={[2.2, 0, 1.8]}>        ← local; world = [2.2, -2, 1.8]
  <Personagem />                           ← coordenadas diretas em world-space
  <EstanteInterativa position={[-4,-2,2]}> ← world-space direto
```

⚠️ **Armadilha principal:** Quarto tem `position={[0,-2,0]}`, mas tudo *dentro* dele usa coordenadas locais. O personagem e EstanteInterativa usam world-space diretamente.

## Tarefa 1: Componente `Debug3D.tsx`

**Arquivos:**
- Criar: `codigo/pecas/Debug3D.tsx`

**O que faz:**
- Lê `window.location.search` para checar se `?debug=1` está na URL
- Usa `useThree()` para acessar a `scene`
- A cada frame (ou em mount), roda `scene.traverse()` e popula `window.__cena3d`
- Renderiza `<axesHelper>` e `<boxHelper>` em cada objeto com nome

**Estrutura do `window.__cena3d`:**
```ts
interface ObjetoDebug {
  nome: string
  tipo: string           // 'Mesh' | 'Group' | 'SkinnedMesh' | etc
  pai: string            // nome do pai direto
  posicaoLocal: [number, number, number]
  posicaoWorld: [number, number, number]
  escala: [number, number, number]
  visivel: boolean
  boundingBoxWorld?: {   // só em Mesh/SkinnedMesh
    min: [number, number, number]
    max: [number, number, number]
    tamanho: [number, number, number]
  }
}
```

**Passo 1: Criar o arquivo**

```tsx
/*
 * Debug3D.tsx — Visualizador de cena 3D para debug de posicionamento
 *
 * Ativar: adicione ?debug=1 na URL (ex: localhost:5173?debug=1)
 * Usar:   no console do browser, rode: copy(window.__cena3d)
 *         depois cole o JSON no chat do Claude para ele entender a cena
 *
 * O que aparece na tela:
 *   - Eixos RGB em cada objeto (X=vermelho, Y=verde, Z=azul)
 *   - Grade no chão (Y=0 do world-space)
 *   - Bounding boxes wireframe em cada mesh
 */

import { useEffect, useRef } from 'react'
import { useThree, useFrame } from '@react-three/fiber'
import { Grid } from '@react-three/drei'
import * as THREE from 'three'

// Declaração global para TypeScript não reclamar de window.__cena3d
declare global {
    interface Window {
        __cena3d: ObjetoDebug[]
    }
}

interface ObjetoDebug {
    nome: string
    tipo: string
    pai: string
    posicaoLocal: [number, number, number]
    posicaoWorld: [number, number, number]
    escala: [number, number, number]
    visivel: boolean
    boundingBoxWorld?: {
        min: [number, number, number]
        max: [number, number, number]
        tamanho: [number, number, number]
    }
}

// Verifica se ?debug=1 está na URL
function modoDebugAtivo(): boolean {
    return new URLSearchParams(window.location.search).get('debug') === '1'
}

// Converte Vector3 para array arredondado com 2 casas decimais
function v3(v: THREE.Vector3): [number, number, number] {
    return [
        Math.round(v.x * 100) / 100,
        Math.round(v.y * 100) / 100,
        Math.round(v.z * 100) / 100,
    ]
}

export function Debug3D() {
    const { scene } = useThree()
    const auxiliaresRef = useRef<THREE.Object3D[]>([])
    const ativo = modoDebugAtivo()

    // Atualiza window.__cena3d a cada 2 segundos (não precisa ser todo frame)
    const ultimaAtualizacao = useRef(0)
    useFrame((state) => {
        if (!ativo) return
        if (state.clock.elapsedTime - ultimaAtualizacao.current < 2) return
        ultimaAtualizacao.current = state.clock.elapsedTime
        atualizarMapaCena()
    })

    function atualizarMapaCena() {
        const mapa: ObjetoDebug[] = []
        const posicaoWorld = new THREE.Vector3()
        const bbox = new THREE.Box3()

        scene.traverse((obj) => {
            // Ignora os próprios helpers de debug para não poluir o JSON
            if (obj.userData.__debugHelper) return
            // Ignora objetos sem nome (intermediários do drei/fiber)
            if (!obj.name) return

            obj.getWorldPosition(posicaoWorld)

            const entrada: ObjetoDebug = {
                nome: obj.name,
                tipo: obj.type,
                pai: obj.parent?.name || 'Scene',
                posicaoLocal: v3(obj.position),
                posicaoWorld: v3(posicaoWorld),
                escala: v3(obj.scale),
                visivel: obj.visible,
            }

            // Bounding box só em meshes (tem geometria)
            if ((obj as THREE.Mesh).isMesh || (obj as THREE.SkinnedMesh).isSkinnedMesh) {
                try {
                    bbox.setFromObject(obj)
                    const tamanho = new THREE.Vector3()
                    bbox.getSize(tamanho)
                    entrada.boundingBoxWorld = {
                        min: v3(bbox.min),
                        max: v3(bbox.max),
                        tamanho: v3(tamanho),
                    }
                } catch {
                    // alguns meshes de shader não têm geometria mensurável
                }
            }

            mapa.push(entrada)
        })

        window.__cena3d = mapa
    }

    // Adiciona helpers visuais (eixos + bbox) na cena
    useEffect(() => {
        if (!ativo) return

        // Limpa helpers anteriores
        auxiliaresRef.current.forEach(h => scene.remove(h))
        auxiliaresRef.current = []

        // Aguarda um frame para modelos GLB carregarem
        const timeout = setTimeout(() => {
            scene.traverse((obj) => {
                if (obj.userData.__debugHelper) return
                if (!obj.name) return

                // Eixos em cada objeto com nome
                const eixos = new THREE.AxesHelper(0.5)
                eixos.userData.__debugHelper = true
                obj.add(eixos)
                auxiliaresRef.current.push(eixos)

                // Bounding box wireframe em meshes
                if ((obj as THREE.Mesh).isMesh) {
                    const boxHelper = new THREE.BoxHelper(obj as THREE.Mesh, 0xffff00)
                    boxHelper.userData.__debugHelper = true
                    scene.add(boxHelper)
                    auxiliaresRef.current.push(boxHelper)
                }
            })
            atualizarMapaCena()
        }, 500)

        return () => {
            clearTimeout(timeout)
            auxiliaresRef.current.forEach(h => {
                if (h.parent) h.parent.remove(h)
            })
            auxiliaresRef.current = []
        }
    // eslint-disable-next-line react-hooks/exhaustive-deps
    }, [ativo, scene])

    if (!ativo) return null

    return (
        <>
            {/* Grade no chão do world-space (Y=0) */}
            <Grid
                args={[20, 20]}
                position={[0, 0, 0]}
                cellSize={1}
                cellColor="#888888"
                sectionSize={5}
                sectionColor="#ff4444"
                sectionThickness={1.5}
                fadeDistance={30}
            />
            {/* Grade no chão do quarto (Y=-2, onde o personagem anda) */}
            <Grid
                args={[10, 10]}
                position={[0, -2, 0]}
                cellSize={1}
                cellColor="#4488ff"
                sectionSize={5}
                sectionColor="#0044ff"
                sectionThickness={1.5}
                fadeDistance={20}
            />
        </>
    )
}
```

# 3D Spatial Context — Trivial/DyCard Room

## When to use this skill

Invoke before ANY of these tasks:
- Moving or repositioning a 3D object
- Fixing a bug where an object is invisible, inside another, or floating
- Adding a new object to the scene
- Debugging why something looks wrong in 3D

## Step 1: Get the scene JSON

Ask the user:
> "Você tem o JSON da cena atual? Se não:
> 1. Abra `localhost:5173?debug=1`
> 2. No console do browser, rode: `copy(window.__cena3d)`
> 3. Cole aqui"

If the user says they don't have it or can't run it, proceed with Step 2 instead.

## Step 2: Map the coordinate system from code

Read `codigo/posicoes.ts` and `codigo/pecas/Quarto.tsx` before touching any coordinates.

Key rule — THE MOST COMMON BUG:
```
<Quarto position={[0, -2, 0]}>        ← group offset: Y is -2 in world
  <Mesa position={[-2.5, 2, -3]}>     ← local Y=2, world Y = 2 + (-2) = 0 ✓
  <Banquinho position={[0, -0.35, 2.4]}> ← local Y=-0.35, world Y = -0.35 + (-2) = -2.35
```

**Always calculate world-space before proposing coordinates:**
```
world_position = local_position + all_parent_offsets
```

## Step 3: Verify bounds before confirming

The room bounds in world-space:
- X: [-4.75, 4.75] (walls at ±4.75)
- Y: [-2.5, 6] (floor at -2, ceiling at ~6)
- Z: [-4.75, 4.75] (walls at ±4.75)
- Floor (where character walks): Y = -2
- Desk surface: Y ≈ 0 (world)

## Step 4: Name objects when creating them

Always add `name` prop to important groups/meshes so they appear in `window.__cena3d`:
```tsx
<group name="MinhaEscrivaninha" position={[...]}>
```

## Common bugs and fixes

| Symptom | Likely cause | How to check |
|---------|-------------|--------------|
| Object invisible | Inside another object or behind wall | Check world-space Y — is it below -2? |
| Object floating | Wrong Y — forgot parent offset | Calculate world Y = local Y + (-2) |
| Object clipping into wall | World X or Z > 4.5 | Check world X/Z bounds |
| Character doesn't reach object | `posicoes.ts` uses world-space, room uses local | posicoes.ts values are world-space |
```

## Como usar o sistema depois de pronto

### Fluxo de debug (você detecta um bug):
1. Abre `localhost:5173?debug=1`
2. Vê visualmente qual objeto está errado (eixos + bbox)
3. No console: `copy(window.__cena3d)`
4. Cola o JSON aqui no chat
5. Claude analisa world-space real e propõe fix correto

### Fluxo de desenvolvimento (Claude vai mexer em posição):
1. Skill `3d-posicoes` ativa automaticamente
2. Claude pede o JSON se não tiver
3. Claude calcula world-space antes de propor coordenadas
4. Claude verifica bounds antes de confirmar

### Atalho rápido (sem abrir debug):
Quando reportar um bug de posição, cole sempre:
```
Objeto: [nome]
Pai: [nome do pai]
Posição local: [x, y, z]
Offset do pai: [x, y, z]
```
Isso é suficiente para o Claude calcular onde o objeto está de verdade.
